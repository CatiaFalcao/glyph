## 1. Objetivo do PoC

Este documento apresenta uma **Prova de Conceito (PoC)** do **GPN — Glyph Problem Network**, utilizando **20 GLYPHs reais**, já validados no repositório oficial do GLYPH Protocol.

O objetivo deste PoC é demonstrar:

- Como GLYPHs podem ser **conectados em rede**
- Como identificar **clusters de problemas**
- Como detectar **padrões sistêmicos recorrentes**
- Como o GPN viabiliza análises quantitativas e estruturais **sem violar o GLYPH Protocol**

> ⚠️ Importante:  
> O GPN **não resolve problemas**.  
> Ele **analisa relações entre problemas formalmente estruturados**.

## 3. Modelo de Conexão do GPN

No GPN, **arestas (edges)** não representam soluções ou causalidade absoluta.  
Elas representam **similaridade estrutural** entre GLYPHs.

### 3.1 Tipos de Arestas Utilizadas

| Tipo de Edge | Critério |
|--------------|---------|
| Shared Variable | Variáveis semanticamente equivalentes |
| Shared Impact | Impactos iguais ou sobrepostos |
| Shared Leverage Point | Pontos de alavancagem equivalentes |
| Causal Pattern | Cadeia causal com estrutura semelhante |
| Governance Pattern | Falhas de governança recorrentes |

## 4. Feature Set Utilizado no GPN

### 4.1 Features Canônicas

## 7. O que o GPN Revela (e o GLYPH isolado não)

- Problemas são **repetidos entre sistemas**
- Falhas estruturais atravessam domínios
- Governança deficiente é um padrão transversal
- Sustentabilidade falha por **problemas mal formulados**

---

## 8. O que o GPN NÃO faz

✔ Não sugere soluções  
✔ Não altera GLYPHs  
✔ Não cria causalidade artificial  
✔ Não viola o GLYPH Protocol
